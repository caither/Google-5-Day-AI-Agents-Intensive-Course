# Design Spec: README Creation

This document outlines the design and implementation details for creating a clean, concise `README.md` at the root of the "Smart Logistics Dispatch Center" project.

## Goal
To write a concise and informative `README.md` in Traditional Chinese (`zh-tw`) that introduces the project and links to the comprehensive documentation in `docs/` without repeating instructions.

## Scope of Changes
Create the `README.md` file in the root directory.

### Structure of README.md
1. **Title**: Smart Logistics Dispatch Center (智慧物流控制中心)
2. **Description**: Overview of the interactive logistics visualization web application.
3. **Key Features**:
   - Leaflet.js map with Esri tiles
   - Smart route planning (Nearest Neighbor)
   - Dynamic path drawing and rotating truck marker
   - Statistics dashboard and completed state toast
4. **Project Layout**: Map of key files and directories.
5. **Documentation & Deployment**:
   - Links to `docs/Cloud-Run-Deployment-Guide.md` (CLI guide)
   - Links to `docs/ref/Deploy from AI Studio to Cloud Run (zh-tw).md` (Codelab reference)

1. Check the README.md content and Markdown links to files.
